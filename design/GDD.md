# Game Design Document (GDD)

## 1. 한 줄 컨셉

> "외국어를 실제 입력하듯 타자하며, 단어와 문장을 적으로 격파하는 스테이지형 격파 게임."

## 2. 장르 / 카테고리

- **장르**: 액션 + 타이핑 + 학습
- **타겟 플랫폼**: 웹 브라우저 (모바일 후순위)
- **플레이어 수**: 싱글 플레이어
- **세션 길이**: 5~15분

## 3. 핵심 경험 (Player Experience)

플레이어는:
1. 스테이지를 선택하고 (EN/JP/ES, 난이도)
2. 화면에 나타나는 단어/문장을 그 언어의 **실제 입력 방식**으로 타이핑하여
3. 적을 격파하고, 미션을 클리어하며
4. 정확도·속도·점수를 보상으로 받는다

## 4. 디자인 기둥

자세한 내용: `design/pillars.md`

1. **입력 방식의 정확성** (Pillar 1, 절대 우선)
2. **격파의 쾌감**
3. **스테이지형 난이도 곡선**
4. **단어/문장의 이중 의미 (적 + 미션)**
5. **메타 진행의 점진성**

## 7. 진행 흐름 (Session)

```
[메인 메뉴]
    │
    ├──> [스테이지 선택] — 언어 / 난이도 / 테마
    │        │
    │        └──> [스테이지 진행] (3~7분)
    │                │
    │                ├── 격파 모드 (단어/문장 격파)
    │                └── 미션 모드 (조건 충족)
    │                │
    │                └──> [스테이지 결과]
    │
    ├──> [연습 모드] — 자유 입력, 미션 없음
    │
    ├──> [통계] — 누적 WPM, 정확도, 언어별 진척
    │
    └──> [설정] — 액센트 모드, 키맵, 색맹 모드
```

## 8. 메타 진행

- **레벨**: 누적 점수 기반
- **해금**:
  - 새 언어 (언어당 기본 1개 스테이지 무료, 추가 스테이지 = 레벨/점수 게이트)
  - 새 테마 (시각/배경)
  - 도전 스테이지 (특수 조건)
- **세이브**: localStorage (Phase 7)

## 9. UI/UX 원칙

- **최소 UI**: 플레이어가 텍스트에 집중하도록
- **즉각 피드백**: 키 입력, 정확/오류 시각/청각 표시
- **시각적 명확성**: 타겟 단어, 현재 입력, 정확도 별도 영역
- **접근성**: 색맹 모드, 폰트 크기 조절, 사운드 on/off

## 11. 성공 지표

- 플레이어가 5분 안에 첫 미션을 클리어한다
- 1주차 리텐션 30%+
- 평균 WPM이 10% 상승 (사용자 보고)
- 모든 언어에서 입력 정확성 95%+ (테스트 통과)

- 음성 인식
- 손글씨/필기 입력
- 멀티플레이어 대전 (단순 경쟁은 가능)
- 게임 내 결제 (free-to-play 기본)
- 게임 내 결제 / 광고

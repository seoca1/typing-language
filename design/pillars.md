# Design Pillars

이 게임의 정체성을 정의하는 5가지 기둥. 모든 디자인 결정은 이 기둥에 비추어 평가된다.

## Pillar 1: 실제 입력 방식의 정확성 (Authentic Input Fidelity)

> "각 언어의 실제 입력 방식을 그대로 살려야 한다"

- EN: 표준 QWERTY 그대로
- JP: 로마자→한자/히라가나 매핑을 정확히 (예: `konnichiwa` → `こんにちは`)
- ES: 액센트 문자 직접 입력 + ASCII 폴백 옵션

**왜 중요한가**: 이 게임의 차별점은 "타자 연습"이 아니라 "외국어를 실제처럼 입력하는 연습"이다. 단순 로마자 무시나 폰트 미적용은 차별점을 죽인다.

**트레이드오프**: 일부 사용자에게는 JP가 어렵게 느껴질 수 있다. → 난이도 곡선과 미션 설명으로 보완.

## Pillar 2: 격파의 쾌감 (Combat-as-Typing Satisfaction)

> "타이핑은 방어의 도구가 아니라 격파의 행위"

- 단어/문장은 화면에서 **적(敵)**으로 등장
- 정확히 타이핑 → 적 처치, 시각/청각 피드백
- 타이핑이 곧 전투 → "공허한 연습"이 아닌 "의미 있는 행동"

**왜 중요한가**: 타자 게임은 지루해지기 쉽다. 격파 시스템은 동기부여를 게임 자체가 제공한다.

**트레이드오프**: 격파 밸런스가 깨지면 답답함 또는 허무함. → placeholder WPM/정확도 기반 점수 공식으로 시작, 점진 조정.

## Pillar 3: 스테이지형 난이도 곡선 (Stage-based Difficulty Curve)

> "한 판이 5~15분, 점점 어려워진다"

- 1~50 스테이지 (확장 가능)
- 초반: 짧은 단어 / 후반: 긴 문장, 속도 제한, 복잡한 문법
- 언어별 특수문자 강제 (예: ES 후반: `ñ`/`á` 강제, JP 후반: 한자 비율 증가)

**왜 중요한가**: 학습 곡선이 명확하면 플레이어가 성취감을 느낀다. 무작위 단어 나열은 학습 효과를 떨어뜨린다.

**트레이드오프**: 너무 가파르면 좌절. → 미션 보상/연습 모드로 보완.

## Pillar 4: 단어/문장의 이중 의미 (Words as Both Enemy and Mission)

> "같은 단어가 적이기도 하고 미션 목표이기도 하다"

- 격파 모드: 단어 = 적 (HP가 글자 수에 비례)
- 미션 모드: 단어/문장 = 클리어 조건 (예: "JP 명사 5개를 정확히 입력하라")
- 미션 = 격파의 **집합** 또는 **제약 조건**이 있는 격파

**왜 중요한가**: 단일 메카닉(격파만)은 단순하고, 단일 목표(미션만)는 게임성이 약하다. 이중화로 깊이를 만든다.

**트레이드오프**: 시스템 복잡도 증가. → Phase 1에서는 단순한 미션으로 시작.

## Pillar 5: 메타 진행의 점진성 (Gentle Meta Progression)

> "매 세션이 조금씩 깊어진다"

- 플레이어 레벨, 해금된 스테이지/언어/테마
- 단, **하드 게이트 없음** — 초보자도 첫 세션에서 모든 언어 플레이 가능
- 메타 진행은 옵션, 강제는 아님

**왜 중요한가**: 한 게임이 끝나면 "다음에 또 하고 싶다"가 핵심. 메타 진행이 너무 강하면 신규 플레이어가 벽을 느낌.

**트레이드오프**: 메타 진행이 약하면 리플레이 가치 감소. → cosmetic 해금 + 도전 스테이지 해금으로 균형.

---

## Pillar 1이 절대 우선

5개 기둥 중 **Pillar 1 (입력 방식 정확성)** 이 절대 우선이다. 게임성이 약하거나 메타 진행이 부족해도, **입력 방식이 정확하지 않으면 이 게임은 실패한 게임**이다. 디자인 충돌 시 Pillar 1을 보호한다.

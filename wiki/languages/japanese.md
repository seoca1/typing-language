# Japanese (JP) - 입력 방식과 코퍼스

> ADR-0002: 로마자→한자 직접 매핑

## 입력 방식

**전략**: 화면에는 한자/히라가나/가타카나로 표시, 사용자는 로마자로 입력.

### 표시 → 입력 예시

| 표시 (Display) | 입력 (Romaji) | 의미 |
| --- | --- | --- |
| こんにちは | konnichiwa | hello |
| ありがとう | arigatou | thank you |
| がっこう | gakkou | school |
| きょう | kyou | today |
| おおさか | oosaka | Osaka |
| とうきょう | toukyou | Tokyo |

### 촉음 (促音) — 다음 자음 더블

|っ| 다음 자음 더블로 입력 |
|---|---|
| がっこう | gakkou (gak**kk**ou) |
| ざっし | zasshi |
| きっぷ | kippu |

### 장음 (長音)

| 표시 | 입력 | 규칙 |
| --- | --- | --- |
| おう | ou | お + う → おう |
| おお | oo | お + お → おお (단어 시작이 아닌 경우) |
| とう | tou | と + う → とう |
| きょう | kyou | き + ょ + う → きょう |

**주의**: `kyou` vs `kiyou` (모음 결합) — 매핑 테이블에 명시.

### ん (n)

| 규칙 | 예시 |
| --- | --- |
| 모음/y 직전 외 모든 n | `konnichiwa` (nn), `sensei` (n + se) |
| 모음/y 직전의 n | `kuni` (ku + ni, ni가 다음), `kin` (ki + n) |

> **규칙**: 다음 글자가 모음(a/i/u/e/o) 또는 y가 아닐 때만 nn으로 처리. 또는 명확성을 위해 매핑 테이블에 명시.

## 코퍼스

### 출처

- **JLPT 단어 목록**: N5~N1 (일본어 능력 시험)
- **빈도순 단어**: 국립국어원 빈도, BCCWJ
- **회화 표현**: 천재교과서, Minna no Nihongo
- **문장**: 위키피디아, NHK Web News

### 코퍼스 형식

```yaml
- id: jp_001
  display: こんにちは
  romaji: konnichiwa
  meaning: hello
  level: 5  # JLPT N5
  category: greeting
```

### 한자 표시 옵션

선택적 기능:
- **히라가나 동시 표시**: `こんにちは (konnichiwa)` 형태로 표기
- **마우스 호버 후리가나**: 캔버스 한계로 Phase 7+
- **단순 한자 모드**: 초급자는 한자만 보고 타이밍

기본값: 단어의 `display`만 표시.

---

## 🎯 테스트 목적

게임 도중 메뉴로 빠져나간 후 다시 게임을 시작할 수 없는 문제를 재현하고 진단

---

## 📋 테스트 시나리오

### **기본 재현 단계:**

1. **페이지 접속**
   ```
   https://seoca1.github.io/typing-language/
   ```

2. **첫 번째 게임 시작**
   - 메인 메뉴에서 "English" 선택
   - 단어/문장 몇 개 타이핑 (최소 3개)
   - 정상 작동 확인

3. **메뉴로 복귀**
   - ESC 키 누르기 또는
   - 화면 상단 "메뉴" 버튼 클릭
   - 메인 메뉴로 돌아가는지 확인

4. **두 번째 게임 시작 (재현)**
   - 다시 "English" 선택
   - **예상:** 게임이 정상 시작되어야 함
   - **실제:** ❓ (테스트 필요)

## 🖥️ 브라우저 콘솔 확인

### **콘솔 열기:**
- **Chrome/Edge:** F12 또는 Ctrl+Shift+I
- **Firefox:** F12
- **Safari:** Cmd+Option+I (개발자 메뉴 활성화 필요)

### **확인할 로그:**

게임 재시작 시 다음 로그가 출력되어야 함:

```
[GameReducer] BACK_TO_MENU: Resetting game state
[GameReducer] Current phase: stage
[GameReducer] Preserving player data: {...}
[GameReducer] New phase: menu
```

다시 시작 시:
```
[GameReducer] START_STAGE: Starting new stage
[GameReducer] Stage: stage-en-1 en
[GameReducer] Enemy: hello
```

### **에러 확인:**

빨간색 에러 메시지가 있다면 전체 복사:
- 에러 메시지
- 스택 트레이스
- 파일명과 줄 번호

## 🐛 예상 원인과 확인 방법

### **원인 1: State 초기화 문제**

**확인 방법:**
```javascript
// 콘솔에서 실행
console.log(window.__REACT_STATE__);
```

**예상 로그:**
- BACK_TO_MENU 후 phase가 'menu'로 변경되었는지
- START_STAGE 후 phase가 'stage'로 변경되었는지

### **원인 2: 이벤트 리스너 중복**

**확인 방법:**
```javascript
// 콘솔에서 실행
getEventListeners(document);
getEventListeners(window);
```

**확인 사항:**
- keydown 리스너가 여러 개 등록되어 있는지
- 각 메뉴 전환 시 리스너 개수 증가하는지

### **원인 3: 애니메이션 프레임 정리 안 됨**

**확인 방법:**
콘솔에서 다음 로그 확인:
```
[Renderer] Starting animation loop
[Renderer] Stopping animation loop
```

두 번째 게임 시작 시 "Starting" 로그가 2번 이상 출력되는지 확인

### **원인 4: 컴포넌트 마운트/언마운트 문제**

**확인 방법:**
React DevTools에서:
- GameScreen 컴포넌트가 올바르게 마운트/언마운트되는지
- useEffect cleanup이 실행되는지

## 컨텍스트

`typing_language` 게임에 한국어(KR)를 추가한다. 일본어(ADR-0002)는 "로마자→한자/히라가나 매핑" 패턴을 채택해 IME 의존 없이 키보드 자판만으로 플레이 가능했다. 한국어도 동일한 패턴을 따를지, 아니면 한국어 특성에 맞는 다른 방식(직접 한글 타이핑, IME 의존 등)을 채택할지 결정해야 한다.

### 제약 / 요구사항

- **IME 비의존** 선호 (ADR-0002, 게임의 핵심 가치: "실제 언어 입력 방식을 정확히 재현")
- **크로스플랫폼** (Windows / macOS / Linux / Web)
- **타이핑 게임 메카닉과 정합** (예측 가능한 키 시퀀스, 학습 가능성)
- **Language 위키(`Language/wiki/Korean/`)와 연계** — 입력 매핑도 Language 위키에서 인용

## 옵션 비교

| --- | --- | --- | --- |

| **D. 영문 표기 → 한글 매핑 (transliteration 외래어)** | 일부 외래어 단어에 자연스러움 (예: "Korea" → "코리아") | 한국어 어휘의 극히 일부만 해당, 일반 어휘 학습에 부적합 | 단독으로는 부적합 |

## 결정 (Accepted)

**옵션 A — 로마자(국립국어원) 직접 입력** 채택.

### 이유

1. **ADR-0002 패턴 일치**: 일본어가 romaji→한자 매핑을 채택한 것과 동일한 철학. 게임의 핵심 가치("실제 입력 방식 재현")는 한국어의 **라틴 문자 표기(로마자)** 가 표준이라는 사실에 의해 뒷받침된다.
2. **IME 비의존**: Web/크로스플랫폼 환경에서 `compositionstart`/`compositionend` 이벤트 처리는 까다롭고, 사용자 시스템의 IME 설정에 따라 동작이 달라진다. 게임은 이 변동을 원하지 않는다.
3. **타이핑 게임 메카닉과 정합**: 키보드의 영문 자판만으로 모든 한국어 단어를 입력할 수 있어, 학습 곡선이 명확하다.
4. **Language 위키와의 연계**: `Language/wiki/Korean/vocabulary/` 페이지에 `display`(한글) + `input`(로마자) 두 필드를 두어 자연스럽게 게임 코퍼스(`raw/kr_words.md`)의 `source` 인용과 결합된다.
5. **Pillar "언어 정확성"**: 국립국어원 로마자 표기법은 한국어의 공식 라틴화 표준으로, 한국어 비사용자에게도 학습 가능한 규칙이다.

### 트레이드오프 인정

- **받침·발음 변동**: ㄷ/ㅂ/ㅅ/ㅎ + 모음의 변동(예: 같이 → gachi, 있습니다 → isseumnida)은 단어별 매핑 테이블로 처리한다. 일본어의 촉음(っ)·장음(ー)·요음 처리와 동일한 부담.
- **로마자 표기법 선택**: 국립국어원(Revised Romanization)을 기본으로 하되, 다른 표기법(Yale, McCune-Reischauer) 혼용은 단어별 `note` 필드에 표기.

## 결과 / 영향

### 긍정적

- 일관된 패턴: EN(직접) / ES(액센트 직접+ASCII 폴백) / JP(romaji→한자) / **KR(romaji→한글)** 모두 IME 비의존
- 핸들러 구현이 JP 핸들러와 유사한 구조(`prefix 매칭 + 단어별 매핑 테이블`)
- Language 위키에서 가져온 vocabulary 페이지 구조(`display` + `input`)를 그대로 활용

### 부정적 / 트레이드오프

- 발음 변동이 많은 단어는 매핑 테이블이 커진다 (일본어 촉음·요음과 같은 부담)
- 한국인 사용자에게는 "타이핑 게임이 한글을 직접 입력 안 한다"는 점이 어색할 수 있음 (온보딩에서 명시 필요)
- 로마자 표기법 혼동 가능 (구개음화, 비음화 등 변동 표기)

### 제약

- 게임에서 `input` 필드는 항상 로마자(ASCII)로 제한. 한글 직접 입력은 지원하지 않음 (옵션 C 미채택)
- 받침 입력은 "발음 기준"으로 통일 (표기 기준과 다를 수 있음). 예: 같이(gachi), 않습니다(animnida)

## 열린 질문

- [ ] **로마자 표기법**: 국립국어원(국어 로마자 표기법) vs. Revised Romanization vs. Yale vs. McCune-Reischauer — 기본값은 국립국어원 / Revised Romanization 제안. 사용자 확정 필요.
- [ ] **발음 변동 깊이**: 자주 쓰는 변동만(연음·비음화·구개음화·경음화 일부) vs. 모든 변동 — 기본값은 "자주 쓰는 변동 + 단어별 매핑". 사용자 확정 필요.
- [ ] **받침 입력**: 발음 기준 vs. 표기 기준 — 기본값은 발음 기준 제안.
- [ ] **스테이지별 입력 모드 토글**: 옵션 C의 변형으로, 일부 스테이지에서만 한글 직접 입력 허용 여부.
- [ ] **한자어 병기**: 단어 페이지에 한자(漢字) 표기 병기할지 (예: 韓國 → 한국).
- [ ] **조사·어미**: 단어와 조사 결합 시 띄어쓰기 / 붙여쓰기 정책. 기본값은 붙여쓰기(안녕하세요 단일 단어).

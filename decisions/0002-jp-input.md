# 0002 - 일본어 입력: 로마자→한자 직접 매핑

## 상태

Accepted (2026-06-18, 사용자 결정)

## 컨텍스트

일본어는 한자/히라가나/가타카나로 표시되지만, 키보드 입력은 로마자로 한다. 게임에서 이 변환을 어떻게 처리할지 결정 필요.

**핵심 요건**:
- Pillar 1: 실제 입력 방식의 정확성
- 게임 친화성: 복잡한 IME 변환 과정이 게임의 재미를 해치지 않아야 함
- 학습 효과: 일본어 학습에 도움이 되어야 함

## 옵션 비교

| 옵션 | 장점 | 단점 | 비고 |
| --- | --- | --- | --- |
| **로마자→한자 직접 매핑** | 게임 친화적, IME 의존성 없음, 단순한 매칭 | 한자 표기 한 가지만 표시 가능 | **선택됨** |
| 실제 IME 변환 시뮬레이션 | 현실적, 한자/히라가나 자동 변환 학습 | 복잡한 구현, 입력 → 후보 → 선택 단계 필요 | |
| 히라가나/가타카나 직접 입력 | 실제 일본어 키보드와 동일 | 한국/일본 사용자만 가능, 글로벌 접근성 낮음 | |
| 한자만 직접 입력 | 가장 짧은 표기 | 일본 사용자에게만 의미, 글로벌 불가 | |

## 결정

**로마자→한자 직접 매핑** (ADR-0002)

```
화면 표시: こんにちは
사용자 입력: konnichiwa
매칭: 정확히 일치 시 처치
```

## 이유

1. **글로벌 접근성**: 한국/일본 사용자 외에는 일본어 키보드를 사용할 수 없음
2. **게임 친화성**: IME 후보 선택 과정 없이 한 번에 입력
3. **Pillar 1 충족**: 일본어 학습자가 실제 사용하는 romaji 입력 방식을 그대로 사용
4. **단순한 구현**: 매핑 테이블 + prefix 매칭으로 충분
5. **학습 효과**: romaji → 한자 매핑을 반복 학습

## 결과 / 영향

### 긍정적
- 한국/중국/일본 모든 사용자가 플레이 가능
- 게임 흐름이 단순 (한 번 입력 = 한 번 결과)
- 단어/문장마다 명확한 정답 romaji

### 부정적 / 트레이드오프
- 한자가 모르는 사용자는 표시된 한자를 읽기 어려움 → **히라가나 표기 옵션** 또는 **후리가나 동시 표시** 검토
- romaji 표기법 차이 (헵번 vs 무로마지 등) → 통일된 표기로 제한
- 한자 다의어 처리 어려움 (예: 今日/きょう) → 단어당 하나의 romaji 매핑 고정

### 제약
- 코퍼스에 romaji 표기 필수 (`raw/jp_words.md` 형식)
- 한자 매핑 테이블 별도 관리
- 표기법 변경 시 코퍼스 전체 갱신

## 구현 메모

### 코퍼스 형식 (`raw/jp_words.md`)

```yaml
- id: jp_001
  display: こんにちは
  romaji: konnichiwa
  meaning: hello
  level: 5  # JLPT N5
  category: greeting
```

### 매핑 규칙

자세한 내용: `design/systems/input-handler.md` > "JapaneseHandler" 섹션
- 장음: `ou`/`oo` → `おう`/`おお`
- 촉음: `kk`, `tt`, `pp` → `っ`
- 요음: `kyo`, `sho`, `cho` 등
- `ん`: 모음/y 직전 외 모든 `n`

### 한자 읽기 어려움 대응 (선택)

- 옵션 A: 히라가나로 동시 표시 (display에 병기)
- 옵션 B: 마우스 호버 시 후리가나 표시
- 옵션 C: JP 한자 포함 단어에 한자 비율 단계적 증가

# 0004 - 렌더링 방식 (Canvas 2D / DOM / WebGL)

## 상태

**Accepted** (2026-06-24)

## 컨텍스트

게임의 메인 렌더링 영역을 어떻게 그릴지 결정. 격파 효과, 텍스트, HUD를 효율적으로 표시해야 함.

**요건**:
- 텍스트 렌더링 품질 (특히 한자/액센트)
- 격파 파티클 효과
- 적 등장/이동 애니메이션
- 성능 (60fps)

## 옵션 비교

| 옵션 | 장점 | 단점 | 비고 |
| --- | --- | --- | --- |
| **Canvas 2D** | 텍스트 렌더링 우수, 단순한 API, IME 호환 | 직접 제어 필요, 적은 라이브러리 도움 | **추천** |
| DOM + CSS | React 통합, 자동 reflow | 격파 파티클에 약함, 성능 한계 | |
| WebGL (PixiJS) | 고성능, 화려한 효과 | 텍스트 렌더링 추가 작업, 격파 게임엔 과함 | |
| WebGL (Three.js) | 3D 가능 | 격파 게임엔 과함, 학습 곡선 | |

## 결정

**Canvas 2D** (메인 렌더링) + React (UI)

- Canvas: 게임 영역 (적, 격파 효과, 입력 표시)
- React: 메뉴, HUD, 결과 화면 (오버레이)

## 이유

1. **텍스트 품질**: 한자/액센트 텍스트가 Canvas 2D에서 매우 잘 렌더링됨
2. **단순성**: Canvas 2D는 표준 API, 의존성 최소화
3. **성능**: 격파 게임의 요구사항(60fps)에 충분
4. **확장성**: 필요시 WebGL로 업그레이드 가능 (Phaser/PixiJS 도입)
